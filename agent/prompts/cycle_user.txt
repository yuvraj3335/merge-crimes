## Requirements for the prompt you generate

The prompt you generate will be passed directly to the Codex CLI running in
full-auto mode on the developer's machine.  It must instruct Codex to do the
following steps IN ORDER:

### STEP A — Read the source of truth
Read all six repo-city docs:
  docs/REPO_CITY_TRACKER.json, docs/REPO_CITY_PRODUCT_VISION.md,
  docs/REPO_CITY_SYSTEM_DESIGN.md, docs/REPO_CITY_ITERATIVE_WORKFLOW.md,
  docs/REPO_CITY_PHASE_EXECUTION_PLAN.md, docs/REPO_CITY_ITERATION_PROMPT.md

### STEP B — Phase gate check (MANDATORY before writing any code)
List every named deliverable for the current phase.
For each one mark it: DONE | PARTIAL | NOT STARTED.
Answer: "Is there any deliverable that changes real user-visible behavior or
backend functionality that is NOT done?"
- If NO → declare the phase DONE in the tracker, set the next phase to
  "in_progress", and implement the first functional slice of the next phase.
- If YES → that deliverable is the only valid target for this cycle.
Do NOT stay in a phase because there are more things that could be polished.

### STEP C — Reject invalid slice targets
The slice target is INVALID if it only changes:
  - UI copy or text strings
  - CSS colors, borders, spacing, or visual tone
  - Tracker JSON metadata with no code change
If the current_slice_recommendation in the tracker is invalid, ignore it.
Find the next incomplete deliverable that changes real behavior instead.

### STEP D — Implement the chosen slice
Implement ONE slice that does at least ONE of:
  - Adds or changes a user-visible game mechanic
  - Adds or changes backend/worker behavior (auth, ingest, data, caching)
  - Adds or changes real data flowing between frontend and worker
  - Fixes the e2e/smoke harness so it actually runs
Do not start additional slices. Do not broaden scope beyond the chosen target.

### STEP E — Validate
Run the appropriate commands:
  - frontend/ changed → cd frontend && npm run build
  - frontend/ .ts/.tsx/.js/.jsx changed → cd frontend && npm run lint
  - worker/ or shared/ contracts changed → cd worker && npm run build
  NEVER run browser:smoke — it is permanently disabled in this sandbox.

### STEP F — Update the tracker
After successful validation, update docs/REPO_CITY_TRACKER.json:
  - Add the completed slice to the phase's completed_slices array.
  - Set current_slice_recommendation to the next FUNCTIONAL slice (not copy/CSS).
  - Update last_updated.
  - If all phase core deliverables are done, set phase status to "done" and
    advance the next phase to "in_progress".
DO NOT update the tracker if validation failed.

### STEP G — Output
End the response with EXACTLY these eight headings:

  Phase:
  Slice attempted:
  Files changed:
  Commands run:
  Results:
  What works now:
  What is blocked:
  Tracker changes Codex made:

The generated prompt must be specific to the current phase, bounded to one
functional slice, and under 700 words.
